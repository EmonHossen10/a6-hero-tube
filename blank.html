Discuss the scope of var, let, and const

  • The var keyword was the original way to declare variables in JavaScript. Variables declared with var have function scope. This means that they can be accessed from anywhere within the function where they are declared, even if they are declared later in the function.
  • The let keyword was introduced in ES6. Variables declared with let have block scope. This means that they can only be accessed from within the block of code where they are declared.
  • The const keyword was also introduced in ES6. Variables declared with const are constants. They cannot be changed after they are declared. const variables also have block scope.

    • Tell us the use cases of null and undefined

  • To represent the absence of a value. For example, if you are trying to access the value of a variable that has not been assigned a value, the value will be undefined.
  • To represent an empty object. The value null can be assigned to a variable to indicate that the variable does not contain an object.
  • To check for the absence of a value. You can use the === operator to check if a variable is equal to null or undefined.
  • To avoid errors. If you are not sure if a variable has been assigned a value, it is better to assign it the value null to avoid errors.

    • What do you mean by REST API?

  • Resource identification: Every resource in a REST API must be uniquely identifiable by a URI (Uniform Resource Identifier).
  • Resource manipulation through representations: The state of a resource can be manipulated by transferring representations of that resource.
  • Self-descriptiveness: The meaning of a resource and its actions should be understandable from the URI alone.
  • Uniform interface: All resources in a REST API should be accessed using the same set of operations.
  • Stateless: A REST API should be stateless, meaning that each request should be independent of any previous requests.
